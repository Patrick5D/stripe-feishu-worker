# Stripe Feishu Worker

Cloudflare Worker that verifies Stripe webhook events and sends selected payment notifications to a Feishu custom group bot.

## Current Deployment

```text
POST https://bingo.thecelesteway.com/gptimage2/stripe/webhook
GET  https://bingo.thecelesteway.com/health
```

Current project:

```text
slug: gptimage2
label: GPT Image 2
Stripe webhook secret: GPTIMAGE2_STRIPE_WEBHOOK_SECRET
```

## Required Secrets

Configure secrets with Wrangler. Do not commit real values.

```bash
npx wrangler secret put GPTIMAGE2_STRIPE_WEBHOOK_SECRET
npx wrangler secret put FEISHU_WEBHOOK_URL
npx wrangler secret put FEISHU_BOT_SECRET
```

Secret meaning:

```text
GPTIMAGE2_STRIPE_WEBHOOK_SECRET
  Stripe endpoint signing secret for /gptimage2/stripe/webhook.
  Value format: whsec_xxx.

FEISHU_WEBHOOK_URL
  Feishu custom group bot webhook URL.

FEISHU_BOT_SECRET
  Feishu custom group bot signing secret.
```

## Events

Custom payment notices are sent for:

```text
checkout.session.completed
invoice.payment_succeeded
```

Other supported notices:

```text
invoice.payment_failed
customer.subscription.created
customer.subscription.updated
customer.subscription.deleted
charge.refunded
```

## Add A New Project

Each project needs three public identifiers and one private Stripe secret:

```text
slug
  URL-safe project id used in the endpoint path.
  Example: erasio

label
  Human-readable name shown in Feishu.
  Example: Erasio

secret env name
  Uppercase Cloudflare secret binding used by the Worker.
  Example: ERASIO_STRIPE_WEBHOOK_SECRET

Stripe endpoint signing secret
  The whsec_xxx generated by Stripe after creating the webhook endpoint.
  Store it with wrangler secret put; never commit it.
```

### 1. Add The Env Binding Type

Edit `src/index.ts`:

```ts
type Env = {
  GPTIMAGE2_STRIPE_WEBHOOK_SECRET: string;
  ERASIO_STRIPE_WEBHOOK_SECRET: string;
  FEISHU_WEBHOOK_URL: string;
  FEISHU_BOT_SECRET: string;
};
```

### 2. Add The Route Mapping

Add a branch in `getSite`:

```ts
function getSite(pathname: string, env: Env): SiteConfig | null {
  if (pathname === "/gptimage2/stripe/webhook") {
    return {
      slug: "gptimage2",
      label: "GPT Image 2",
      stripeWebhookSecret: env.GPTIMAGE2_STRIPE_WEBHOOK_SECRET,
    };
  }

  if (pathname === "/erasio/stripe/webhook") {
    return {
      slug: "erasio",
      label: "Erasio",
      stripeWebhookSecret: env.ERASIO_STRIPE_WEBHOOK_SECRET,
    };
  }

  return null;
}
```

### 3. Deploy The Code

```bash
npm run typecheck
npx wrangler deploy
```

### 4. Create The Stripe Endpoint

In the matching Stripe account, create a webhook endpoint:

```text
https://bingo.thecelesteway.com/erasio/stripe/webhook
```

Subscribe at least:

```text
checkout.session.completed
invoice.payment_succeeded
invoice.payment_failed
customer.subscription.deleted
charge.refunded
```

Copy the generated endpoint signing secret:

```text
whsec_xxx
```

### 5. Store The Stripe Secret

```bash
npx wrangler secret put ERASIO_STRIPE_WEBHOOK_SECRET
```

Paste the `whsec_xxx` value when Wrangler prompts.

### 6. Verify

Health check:

```bash
curl -sS https://bingo.thecelesteway.com/health
```

Expected:

```json
{"ok":true,"service":"stripe-feishu-worker"}
```

Unsigned webhook requests should fail:

```bash
curl -i -X POST https://bingo.thecelesteway.com/erasio/stripe/webhook
```

Expected:

```text
400 Missing Stripe signature
```

Then verify from Stripe Dashboard webhook attempts. A successful delivery returns HTTP 200 and sends a Feishu message.

## Naming Rules

Use stable names. Do not rename an existing slug unless the Stripe endpoint URL is also changed.

```text
slug: lowercase words, numbers, hyphens only
path: /{slug}/stripe/webhook
secret: {SLUG}_STRIPE_WEBHOOK_SECRET, with hyphens converted to underscores
label: product name shown in Feishu
```

Example:

```text
site: seedance25-ai.com
slug: seedance25
path: /seedance25/stripe/webhook
secret: SEEDANCE25_STRIPE_WEBHOOK_SECRET
label: Seedance25
```

## Security

- Do not commit `whsec_xxx`, Stripe API keys, Feishu webhook URLs, or Feishu signing secrets.
- Store runtime secrets only with `wrangler secret put`.
- The public endpoint is safe to expose because Stripe signatures are verified before notifications are sent.
- Test and live Stripe endpoints have different `whsec_xxx` values. Use separate paths or separate Workers if both must be active.
